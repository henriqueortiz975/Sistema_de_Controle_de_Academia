# Trabalho Prático — Projeto Integrador de Sistema Web

## Projeto Integrador

### Tema

Desenvolvimento de um Sistema Web de Gestão.

---

## Objetivo

Desenvolver uma aplicação web completa utilizando:

- HTML5;
- CSS3;
- JavaScript;
- LocalStorage.

O sistema deverá possuir interface moderna, organizada e responsiva.

---

## Sistema de Controle de Academia

integrantes: Henrique Ibarra Ortiz Urbano e Arthur da Silva Veeck

Funcionalidades mínimas:

- Cadastro de alunos;
- Cadastro de planos;
- Controle de pagamentos;
- Controle de frequência;
- Relatórios.

---

## Requisitos obrigatórios

o projetos deverão possuir os itens abaixo.

### 1. Tela inicial

- Nome do sistema;
- Logo;
- Menu de navegação;
- Apresentação breve do objetivo do sistema.

### 2. Área de cadastro

O sistema deverá permitir, no mínimo:

- Inclusão de registros;
- Alteração de registros;
- Exclusão de registros;
- Pesquisa de registros.

### 3. Persistência de dados

O sistema deverá utilizar LocalStorage para armazenar os dados no navegador.

### 4. Interface

A interface deverá ser:

- Responsiva;
- Organizada;
- Moderna;
- Com paleta de cores definida;
- Com uso de ícones, quando necessário.

### 5. JavaScript

O projeto deverá utilizar:

- Arrays;
- Objetos;
- Funções;
- Eventos;
- Condicionais;
- Estruturas de repetição;
- Manipulação do DOM.

---

## Possibilidade de uso com Node.js

A dupla poderá optar por desenvolver uma versão com Node.js, desde que o sistema possua:

- Servidor básico;
- Rotas organizadas;
- Separação de arquivos;
- Interface HTML, CSS e JavaScript;
- Dados armazenados em arquivo JSON ou estrutura equivalente.

O uso de Node.js não é obrigatório, mas poderá ser utilizado como diferencial técnico.

---

## Entregáveis

### 1. Protótipo

Desenho das telas principais, podendo ser feito em papel, Figma ou outra ferramenta de prototipação.

### 2. Documento de requisitos

O documento deverá conter, no mínimo:

- Objetivo do sistema;
- Público-alvo;
- Requisitos funcionais;
- Requisitos não funcionais;
- Regras de negócio básicas.

### 3. Sistema funcionando

O sistema deverá ser apresentado em funcionamento.

Preferencialmente, o projeto deverá ser publicado no GitHub Pages quando utilizar apenas HTML, CSS e JavaScript.

Caso utilize Node.js, a dupla deverá apresentar o funcionamento localmente ou em serviço de hospedagem compatível.

### 4. Código-fonte

O repositório no GitHub deverá conter:

- Arquivos HTML;
- Arquivos CSS;
- Arquivos JavaScript;
- README explicando o projeto;
- Organização clara das pastas.
